Backbone.Stage

In a Backbone+ application there is one special extended scene that represents the stage. The stage will help with the task of managing your application's viewport properties and current scene.

Creating a Backbone.Stage

Most likely you will want to create your stage after DOM ready has fired. Also, you will want to attach a new router instance to the stage. This way the stage can help with scene changes.

// App Ready
$(document).ready(function() {
	App = new Backbone.Stage();
	App.router = new AppRouter();
});

Functions

render()

Inherited from Backbone.View. The default implementation of render is a no-op. Override this function with your own logic.

resize(w, h)

Resizes the internal stage coordinates. Typically this will be called from a window.resize event callback. Example

$(window).on('resize', function() {
   stage.resize(w, h);		
});
changeScene(scene, name)

Handles processing and changing the current scene. Most likely this method will be called from within a Backbone.Router.

var scene = new MainScene();
stage.changeScene(scene, 'main');
currentScene()

Returns the current scene.

var scene = stage.currentScene();
currentSceneName()

Returns the current scene name.

var sceneName = stage.currentSceneName();
stageWidth()

Returns the current scene width.

var width = stage.stageWidth();
stageHeight()

Returns the current scene height.

var width = stage.stageHeight();
stageOrientation()

Returns the current scene orientation. Returns either portrait or landscape.

var landscape = stage.stageOrientation();
calcWidth(pct)

Returns a percentage width based on stageWidth.

// Returned width will be 50% of the stage width
var width = stage.calcWidth(0.5);
calcHeight(pct)

Returns a percentage height based on stageHeight.

// Returned height will be 50% of the stage height
var height = stage.calcHeight(0.5);
